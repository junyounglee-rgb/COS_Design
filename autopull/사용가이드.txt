================================================================
  Auto Pull - 사용 가이드
================================================================

여러 브랜치를 선택해서 한 번에 풀 받는 도구입니다.
cos-data, cos-common, cos-client 등 여러 저장소를 동시에 처리합니다.

----------------------------------------------------------------
  최초 설치
----------------------------------------------------------------

1. install.bat 더블클릭
   → Python 환경 + 패키지 자동 설치 (1~3분)
   → "Installation complete!" 메시지 확인

2. run.bat 더블클릭
   → 브라우저에서 http://localhost:8504 자동 오픈

----------------------------------------------------------------
  설정 (최초 1회)
----------------------------------------------------------------

1. ⚙️ 설정 펼치기

2. 메인 저장소 경로
   → D:\COS_Project\cos-data (기본값 설정됨)

3. 함께 풀 받을 저장소 (한 줄에 하나)
   → D:\COS_Project\cos-common
   → D:\COS_Project\cos-client

4. 브랜치 목록 (한 줄에 하나)
   → main
   → release_helsinki2
   → release_helsinki3
   → release_helsinki_vng

5. 💾 저장 클릭

----------------------------------------------------------------
  매번 사용법
----------------------------------------------------------------

STEP 1: run.bat 실행

STEP 2: 풀 받을 브랜치 체크박스 선택/해제

STEP 3: [📡 상태 확인] 클릭 (선택사항)
   → 각 브랜치별 원격 업데이트 여부 표시
   → ✅ 최신 / 🔄 N커밋 대기 / ⚠️ 원격 없음

STEP 4: [🔽 풀 받기] 클릭

STEP 5: 진행 상황 로그 확인
   → 각 저장소별 checkout → pull 결과 표시

STEP 6: 결과 확인
   → ✅ 성공 / ❌ 실패 브랜치 요약

----------------------------------------------------------------
  주의사항
----------------------------------------------------------------

- 풀 받는 도중 창을 닫지 마세요
- 작업 완료 후 원래 브랜치로 자동 복귀합니다
- sibling 저장소에 해당 브랜치가 없으면 자동 건너뜀
- 설정은 autopull_branches.yaml에 저장됩니다

----------------------------------------------------------------
  트러블슈팅
----------------------------------------------------------------

Q. "Environment not set up" 오류
→ install.bat을 먼저 실행해주세요.

Q. install.bat 실행 시 Python 환경 생성 실패
→ https://www.python.org/downloads/ 에서 Python 3.12 설치 후 재시도

Q. pull 실패
→ 네트워크 연결 확인
→ 로컬에 커밋되지 않은 변경사항이 있으면 먼저 정리

Q. checkout 실패
→ 로컬에 수정된 파일이 있으면 stash 또는 commit 후 재시도
