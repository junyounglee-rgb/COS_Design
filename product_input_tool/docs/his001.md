## 개발 배경

- 기존: 일반상품(general_store)만 상점상품 탭에서 item-input-template.xlsx로 관리
- 목적: 6개 전체 상품 타입 자동화 + keywords/items 동적 연동
- 결정: Excel 템플릿 유지 (탭-per-타입) + apply 스크립트가 복잡도 전담

## 주요 개발 단계

| 단계 | 내용 | 결과 |
|------|------|------|
| 1 | gen-product-template.py 초안 작성 | 참조탭 5개 + 입력탭 6개, 11탭 템플릿 생성 |
| 2 | 출석패키지 탭 수정 | 즉시지급(instant_items) 제거 (실제 시트에 없음 확인) |
| 3 | 스텝상품 수정 | STEP_COUNT=3, STEP_ITEMS=4 (실제 시트 구조 반영) |
| 4 | $filter 드롭다운 추가 | keywords.xlsx build 시트 → 빌드 ID 목록 E열 저장 → 드롭다운 연결 |
| 5 | apply-product-input.py 작성 | 6개 타입 전체 product.xlsx 삽입 로직 구현 |
| 6 | 슬래시 커맨드 등록 | /product-gen, /product-apply |
| 7 | item-input-template.xlsx 수정 | 상점상품 탭 제거 |

## 주요 발견 사항

### 출석패키지 (attendance_product)

- 실제 시트에 items 컬럼 없음 → 보상은 attendance_product_rewards 시트에만 존재
- 초안에 포함된 instant_items 블록 제거

### 스텝상품 (step_products)

- 실제 시트: 3스텝 × 아이템 4개 (id+qty) + 스텝별 가격 1세트
- 초안: STEP_COUNT=5, STEP_ITEMS=2 → 실제 구조로 수정

### 조건부상품 (conditional_products)

- price %param6 없음 (최대 %param5까지)
- apply 스크립트에서 PRICE_P6_COL 스킵

### $filter 빌드 드롭다운

- keywords.xlsx/build 시트 row4+에서 ID 로드
- 값 앞에 $$ 접두사 붙여 저장 (예: $$NONE, $$DEV, $$LAUNCH_0)
- 📖 키워드 탭 E열에 기입 → 입력 탭 $filter 컬럼에 드롭다운 연결
