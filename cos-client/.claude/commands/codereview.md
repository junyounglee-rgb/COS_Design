건설적으로 피드백을 제공하고, 개선을 위한 구체적인 제안을 하세요. 코드 변경을 제안할 때는 GitHub의 제안 형식을 사용하세요.
CLAUDE.md 파일의 프로젝트 가이드라인을 따르세요.

## 리뷰 가이드라인:
1. 긍정적인 변경사항도 언급하여 균형잡힌 리뷰를 제공하세요
2. 성능, 보안, 가독성, 유지보수성 측면에서 종합적으로 검토하세요
3. Unity 특화 베스트 프랙티스와 모바일 게임 최적화 관점을 고려하세요
4. 메모리 누수, GC Alloc, 프레임 드롭 등 모바일 성능 이슈를 우선적으로 검토하세요
5. null 참조, 배열 경계, 동시성 문제 등 런타임 오류 가능성을 확인하세요
6. **필수 요구사항**: 각 개선 제안마다 반드시 다음 형식으로 제시하세요:
   ```
   // 현재 코드 (문제가 있는 부분)
   [현재 코드]
   
   // 개선된 코드 (수정 후)
   [개선된 코드 5-15줄 분량]
   ```
   개선된 코드가 없는 제안은 절대 허용되지 않습니다.
7. 파일별, 메서드별로 구체적인 라인 번호와 함께 문제점을 지적하세요
8. 성능 측정 가능한 부분은 예상 개선 효과를 수치로 제시하세요 (예: "GC 할당 50% 감소 예상")
