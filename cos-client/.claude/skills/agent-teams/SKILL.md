# Agent Team 스킬

에이전트 팀을 생성하고 팀원들이 병렬로 작업을 수행하도록 조율하는 스킬입니다.

## 역할 구조

- **팀 리더 (나 = 메인 에이전트)**: 팀을 생성하고, 태스크를 할당하며, 팀원의 작업 결과를 리뷰합니다. 팀원들의 알림을 자동으로 수신합니다.
- **팀원 (서브 에이전트)**: Sonnet 모델로 실행되며, 할당된 태스크를 수행합니다. 파일 변경 전에 사용자(사람)의 plan approval을 받아야 합니다.
- **사용자 (사람)**: 팀 구성을 승인하고, 팀원의 plan approval 요청을 승인/거부합니다.

## 워크플로우

### 1단계: 사용자 인터뷰

팀을 생성하기 전에 반드시 사용자에게 다음을 확인하세요:

**수행할 작업:**
- "어떤 작업을 팀으로 수행하고 싶으신가요?"
- 작업의 범위와 목표를 명확히 파악하세요.

**팀 구성 제안 (필수):**
- 작업 내용을 파악했으면, 반드시 팀 구성을 먼저 제안하세요. 사용자에게 팀원 수나 역할을 묻지 말고, 작업을 분석하여 직접 제안합니다.
- 제안 시 각 팀원의 이름, 역할, 담당 범위를 구체적으로 명시하세요.
- 제안 형식 예시:
  ```
  작업을 분석해보니 다음과 같은 팀 구성을 제안드립니다:
  - analyzer (분석 담당): 기존 코드 분석 및 영향 범위 파악
  - implementer-1 (구현 담당 1): 모듈 A 구현
  - implementer-2 (구현 담당 2): 모듈 B 구현

  이 구성으로 진행할까요, 아니면 팀원 수나 역할을 변경하고 싶으신가요?
  ```
- 사용자가 승인하면 바로 팀을 생성하고, 변경을 원하면 반영 후 다시 확인합니다.

### 2단계: 팀 생성

사용자의 답변을 바탕으로 TeamCreate를 사용하여 팀을 생성합니다:

```
TeamCreate:
  team_name: "<작업에-맞는-팀-이름>"
  description: "<작업 설명>"
```

### 3단계: 태스크 생성

TaskCreate를 사용하여 작업을 태스크 단위로 분할합니다. 각 태스크는:
- 명확한 범위와 완료 조건을 가져야 합니다
- 의존성이 있는 경우 blocked_by로 명시합니다
- 아직 owner를 지정하지 않습니다 (팀원 생성 후 할당)

### 4단계: 팀원 생성 (Agent 도구 사용)

각 팀원을 Agent 도구로 생성할 때 반드시 다음 규칙을 따르세요:

#### 필수 설정

1. **모델**: 모든 팀원은 반드시 `model: "sonnet"`을 사용합니다.
2. **권한 모드**: 모든 팀원은 반드시 `mode: "plan"`을 사용합니다. 이렇게 하면 팀원이 파일을 변경하기 전에 사용자의 plan approval을 받게 됩니다.
3. **팀 이름**: `team_name`에 생성한 팀 이름을 지정합니다.
4. **이름**: `name`에 역할을 나타내는 이름을 지정합니다.

#### 팀원 생성 예시

```
Agent:
  name: "frontend-dev"
  description: "프론트엔드 구현"
  model: "sonnet"
  mode: "plan"
  team_name: "my-team"
  prompt: |
    당신은 팀의 frontend-dev 역할입니다.

    [작업 내용 상세 설명]

    작업 절차:
    1. TaskList로 할당된 태스크를 확인하세요.
    2. 태스크를 수행하세요.
    3. 완료되면 TaskUpdate로 태스크를 완료 처리하세요.
    4. TaskList를 다시 확인하여 추가 작업이 있는지 확인하세요.
```

### 5단계: 태스크 할당 및 모니터링

- TaskUpdate로 각 팀원에게 태스크를 할당합니다
- 팀원들의 메시지는 자동으로 전달됩니다
- 팀원이 idle 상태가 되는 것은 정상입니다 — 메시지를 보내면 다시 활성화됩니다
- 필요시 SendMessage로 팀원에게 추가 지시를 보냅니다

### 6단계: 팀 리더 리뷰

팀원이 태스크를 완료하면, 팀 리더(나)가 변경 내용을 리뷰합니다:
- 팀원이 수정한 파일을 읽고 코드 품질, 일관성, 의도한 동작 여부를 확인합니다
- 문제가 있으면 SendMessage로 팀원에게 수정을 요청합니다
- 리뷰 결과를 사용자에게 보고합니다

### 7단계: 완료 및 팀 종료

모든 태스크가 완료되고 리뷰를 통과하면:
1. 최종 결과를 사용자에게 보고합니다
2. SendMessage로 각 팀원에게 `type: "shutdown_request"`를 보내 팀을 종료합니다

## 핵심 규칙 요약

| 항목 | 값 | 이유 |
|------|-----|------|
| 팀원 모델 | `sonnet` | 빠른 응답과 비용 효율성 |
| 팀원 권한 모드 | `plan` | 변경 전 사용자 승인 필수 — 의도치 않은 변경 방지 |
| 팀 생성 도구 | `TeamCreate` | 팀 설정 파일과 태스크 리스트를 자동 생성 |
| 팀원 생성 도구 | `Agent` (with team_name) | 팀에 소속된 에이전트로 생성 |
