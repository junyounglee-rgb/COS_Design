# 커밋 로그 컨벤션

## 1. 양식

```
<type>(<scope>): <제목>

- 상세 내용

Version: v03
Wrike: #12345
```

---

## 2. type 종류

| type | 용도 |
|------|------|
| `feat` | 새로운 기능 추가 |
| `fix` | 버그 수정 |
| `hotfix` | 긴급 버그 수정 |
| `data` | xlsx/pb 데이터 변경 |
| `refactor` | 코드 구조 변경 |
| `docs` | 문서 수정 |
| `chore` | 빌드, 설정 변경 |

---

## 3. scope 란?

> **"이 커밋이 프로젝트의 어느 파트에 영향을 주는가"**

팀원이 로그만 봐도 자기 담당 영역 커밋만 빠르게 필터링할 수 있음

```bash
git log --grep="(monster)"   # 몬스터 관련 커밋만 조회
git log --grep="Version: v03"  # v03 관련 커밋만 조회
git log --grep="Wrike: #12345" # 특정 일감 커밋만 조회
```

### scope 예시 (데이터 카테고리)

| scope | 의미 |
|-------|------|
| `monster` | 몬스터 데이터 |
| `item` | 아이템 데이터 |
| `quest` | 퀘스트 데이터 |
| `balance` | 밸런스 수치 |
| `char` | 캐릭터 데이터 |
| `script` | 변환 스크립트 |
| `ci` | 파이프라인 설정 |

---

## 4. Version / Wrike 태그

| 태그 | 효과 |
|------|------|
| `Version: v03` | 어느 릴리즈용 작업인지 즉시 파악 |
| `Wrike: #12345` | 커밋 ↔ 일감 양방향 추적 가능 |

> 두 태그 모두 **필수** 입력

---

## 5. 실제 예시

### 데이터 수정

```
data(char): 전사 hp 수정

- v02 전용 hp 1000 → 1200 조정

Version: v02
Wrike: #12345
```

### 신규 데이터 추가

```
data(monster): 보스 몬스터 3종 추가

- 드래곤, 트롤, 골렘 기본 스탯 정의
- target: v03, v04 적용

Version: v03
Wrike: #12346
```

### 핫픽스

```
hotfix(item): 무기 드롭 버그 수정

- 희귀 등급 드롭율 오류 수정

Version: v01
Wrike: #12347
```

---

## 6. 커밋 분리 기준

| 상황 | 방법 |
|------|------|
| 서로 다른 Wrike 일감 | 무조건 커밋 분리 |
| 같은 일감, 연관된 작업 | 같이 커밋 가능 |
| 같은 일감, 무관한 작업 | 분리 권장 |

> **하나의 커밋 = 하나의 이유**
> 롤백 시 의도하지 않은 데이터까지 되돌아가는 상황을 방지
