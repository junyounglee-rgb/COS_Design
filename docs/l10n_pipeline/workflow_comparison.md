# 번역 워크플로우 비교: Lokalise → Google Sheet

> 비교 대상: `translation_pipeline.md` (현재) vs `google_sheet_workflow.md` (개선)

---

## 전체 요약

| 항목 | 현재 (Lokalise) | 개선 (Google Sheet) | 감소폭 |
|------|----------------|-------------------|--------|
| **총 작업 단계** | 17단계 | 4단계 | **-13단계** |
| **PM 수작업** | 11단계 | 0단계 | **-100%** |
| **1회 PM 작업 시간** | 46분 | 0분 | **-46분** |
| **작업자 대기 시간** | PM 처리 시점까지 대기<br>(즉시 처리 요청 시 PM 본업 중단) | 최대 5분 | 대기 + 본업 중단 동시 해소 |
| **실제 운영 방식** | 특정 시간 배치 처리<br>(수정 시 다음 배치까지 대기 반복) | 자동 (5분 cron) | 배치 주기 의존 제거 |
| **단일 장애점** | PM 부재 = 전체 중단 | 없음 | 해소 |

## 현재 (Lokalise 방식)

```
┌────────────────────────────────────────────────┐
│ 기획자                                          │
│ ─────────────────────────────────────          │
│ 1. translations.xlsx에 한글 직접 입력           │
└──────────────────┬─────────────────────────────┘
                   ↓
        ⏳ PM 처리 대기
           (즉시 처리 가능하나 본업 중단 →
            보통 특정 시간에 모아서 배치 처리)
                   ↓
┌────────────────────────────────────────────────┐
│ PM — 수동 반복 작업 (46분/회)                   │
│ ─────────────────────────────────────          │
│ 2.  Fork 실행 + cos-data 최신화                 │
│ 3.  번역키 추출 버튼 클릭 + 20초 대기           │
│ 4.  exports/ 확인 및 검수 (오탈자/누락)         │
│ 5.  7종 카테고리 수동 분류                      │
│     (dialog/cookie/quests/ui/item/skill/etc)   │
│ 6.  Lokalise 접속 + 프로젝트 선택               │
│ 7.  카테고리별 파일 개별 업로드                 │
│     ⚠ 설정값 매번 수동 입력                     │
│     ⚠ Differentiate keys by file 해제 (실수위험)│
│ 8.  Skipped 키 처리 → 재업로드                  │
│ 9.  '로컬 키' 작업 탭 등록                      │
│ 10. 기획팀/PM팀 Slack 공지                      │
└──────────────────┬─────────────────────────────┘
                   ↓
┌────────────────────────────────────────────────┐
│ 번역팀                                          │
│ ─────────────────────────────────────          │
│ 11. Lokalise 로그인 + 프로젝트 선택             │
│ 12. 필터 설정 + 번역 대상 파악                  │
│ 13. 번역 입력                                   │
└──────────────────┬─────────────────────────────┘
                   ↓
┌────────────────────────────────────────────────┐
│ PM — 수동 Sync (배치 시점)                      │
│ ─────────────────────────────────────          │
│ 14. Sync 페이지 접속 + "Just Do It" 클릭        │
│ 15. Sync 처리 시간 (약 10분)                    │
│ 16. Fork 확인 → 자동 push 완료 확인             │
│ 17. 완료 공지                                   │
└──────────────────┬─────────────────────────────┘
                   ↓
          [게임 반영 완료]
                   ↓
        ❓ 수정 사항 발견?
          ├─ YES → 기획자 수정 → 다시 ⏳ 대기 (루프)
          └─ NO  → 작업 종료
```

> **핵심 문제**: PM은 즉시 처리도 가능하지만, 그때그때 처리하면 본업을 할 수 없음 →
> 보통 **특정 시간에 모아서 배치 처리**함.
> 기획자가 수정을 발견해도 → 다음 배치 주기까지 또 대기 → 재배포 루프 반복.
> 수정이 여러 번 반복되면 하루 이상 반영이 지연될 수 있음.

## 개선 (Google Sheet 방식)

```
┌────────────────────────────────────────────────┐
│ 기획자                                          │
│ ─────────────────────────────────────          │
│ 1. Google Sheet에 str_id + ko + tag=new 추가    │
└──────────────────┬─────────────────────────────┘
                   ↓
        ┌──────────────────────────┐
        │ CI 자동 (5분 cron)       │
        │ hash 비교 → ko CSV 생성  │
        │ → cos-client 배포        │
        │ → Unity Editor 즉시 반영 │
        └──────────┬───────────────┘
                   ↓
┌────────────────────────────────────────────────┐
│ 번역팀 (Slack 자동 알림 수신)                   │
│ ─────────────────────────────────────          │
│ 2. 워크벤치 툴 접속                              │
│ 3. tag=new/re 필터 → 번역 입력                   │
│ 4. 저장 (tag=done 자동 변경)                     │
└──────────────────┬─────────────────────────────┘
                   ↓
        ┌──────────────────────────┐
        │ CI 자동 (새벽 4시)        │
        │ Google Sheet 전체 읽기    │
        │ → 전체 언어 pb 생성       │
        │ → Portal 업로드           │
        │ → Slack 완료 알림         │
        └──────────┬───────────────┘
                   ↓
          [게임 반영 완료]
```

## 사라지는 작업

| 사라지는 작업 | 어떻게 해결되는가 |
|-------------|-----------------|
| PM 배치 주기 대기 | 5분 CI cron (작업자 즉시 확인) |
| 수정 시 재배포 루프 | 수정 즉시 반영 (루프 자체 소멸) |
| Fork 실행/최신화 | 불필요 (Google Sheet 직접 입력) |
| 번역키 추출 + 20초 대기 | 자동 감지 |
| 7종 카테고리 수동 분류 | 분류 자체가 불필요 (tag 컬럼으로 대체) |
| Lokalise 수동 업로드 | 사라짐 |
| 설정값 수동 입력 | 사라짐 (실수 위험 제거) |
| Skipped 키 처리 | 사라짐 |
| Sync 수동 실행 | CI cron 자동화 |
| 완료 확인 수동 | Slack 자동 알림 |
| PM 직접 번역팀 연락 | Slack 자동 알림 |
| Lokalise 계정 관리 | Google Sheet 공유로 대체 |

## 속도 비교

```
현재 (1) — PM 즉시 처리 요청 시:
  기획 수정 ──▶ PM 본업 중단 ──▶ PM 46분 작업 ──▶ 게임 반영
              ───────────────────────────────────────
              │ 문제: PM이 그때그때 처리하면 본업 불가 │
              ───────────────────────────────────────

현재 (2) — 실제 운영 (특정 시간 배치 처리):
  기획 수정 ──▶ ⏳ 다음 배치 시간 대기 ──▶ PM 46분 작업 ──▶ 반영
              └────────────────────────────────────────┘
              │ 대기 시간: 배치 주기에 의존              │
              │ (하루 1~2회 배치라면 최대 24h+)          │
              └────────────────────────────────────────┘

현재 (3) — 수정 반복 시:
  1차 수정 ──▶ ⏳ 배치 대기 ──▶ PM 작업 ──▶ 반영 ──▶ 오류 발견
                                                       ↓
  2차 수정 ──▶ ⏳ 배치 대기 ──▶ PM 작업 ──▶ 반영 ──▶ 오류 발견
                                                       ↓
  3차 수정 ──▶ ⏳ 배치 대기 ──▶ PM 작업 ──▶ 반영 ──▶ ...
              └─────────────────────────────────────────┘
              │ 수정 횟수 × 배치 주기만큼 지연 누적      │
              └─────────────────────────────────────────┘

개선:
  기획 수정 ──▶ ⏱ 최대 5분 ──▶ Unity 반영 (ko만)
                             └─▶ 번역팀 자율 작업 ──▶ 새벽 4시 ──▶ 전체 언어
              ─────────────────────────────────────────
              │ PM 개입: 0분 / 수정 반복 시에도 5분     │
              ─────────────────────────────────────────
```

## 역할 변화 요약

| 역할 | 현재 | 개선 |
|------|------|------|
| **기획자** | translations.xlsx 한글 입력 | Google Sheet에 str_id+ko+tag 입력 |
| **PM** | 번역 파이프라인 전담 (46분/회) | 관여 없음 |
| **번역팀** | Lokalise 계정 필요, PM 대기 | Google Sheet 직접 접근 |
| **기획 팀장** | - | 긴급 배포 버튼 (선택 사항) |
| **CI** | - | 자동 배포 (5분/새벽 4시) |
