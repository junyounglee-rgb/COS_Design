## 메타-원인 (1~2개)

- **M1: 진실의 원천(Source of Truth)이 분기되어 있다**
  - 같은 키의 한국어 원문이 `translations.xlsx` / `Lokalise rev` / `dictionary(.pb)` 3곳에 각자 저장되고, 각각 **독립적으로 쓰기 가능**.
  - 5건 중 4건(02-05, 03-07, 03-11, 04-17)에서 "어느 값이 진짜인가" 판정 불가 상태가 발생.
  - rev 누적, 대표키 병합, publish 누락, 바이너리 잔여물 — 표면 증상이 달라도 **동일 구조 결함의 다른 얼굴**.

- **M2: 원문 수정의 공식 경로가 유일한 경로보다 비용이 비싸다**
  - 공식 경로 = Excel → datasheet → 업로드 → Sync (기획자 5단계 + 담당자 소환).
  - 편법 경로 = Lokalise 웹에서 한 글자 고치기 (3초).
  - 두 경로의 **비용 격차가 수백 배**일 때 편법은 규율로 막히지 않음. 3/26 지희의 Lokalise 직수정은 개인 실수가 아니라 **비용 최적화의 합리적 선택**이었고, 04-17 재오염의 방아쇠가 됨.
  - 02-20 지혜 PM, 03-11 조경주 PM, 04-17 이경우 PM — 숙련자도 동일 패턴 반복.

> **변경 (v1→v2) 각주 [M2-a]**: 본 항목의 "3초 수렴"은 공식 경로의 **종단 시간(end-to-end)**을 뜻한다. '수동 단계 수 = 0'을 의미하는 것이 아니다. 수동 단계가 0이어도 비동기 대기가 10분이면 M2는 미충족으로 판정한다. Architect가 "수동 단계 0 → M2 만족"으로 자의 축소하는 것은 정의 위반이며, iteration-contract v2 감점 트리거에 해당한다.

## 해결 원칙 (4~6개)

> "규율/교육/주의/혼내기"에 의존하지 않고 **구조로 편법 경로를 소멸**시키는 원칙.

1. **단일 쓰기 원천 (Single Writable Origin for KO)**
   - 한국어 원문은 **정확히 한 곳**에서만 쓰기 가능해야 하며, 나머지 위치는 **읽기 전용 또는 파생물**로 전락시킨다.
   - "어디가 원천인가"가 의심되는 순간이 0이어야 한다. 판정 불가 상태(M1) 자체가 재현 불가.

2. **편법 경로 비용 역전 (Reverse the Cost Gradient)**
   - 공식 경로를 편법 경로보다 **빠르거나 최소한 동등**하게 만든다. 발견자 ≠ 수정자일 때 드는 "담당자 소환 비용"을 구조적으로 제거.
   - 3초짜리 편법이 존재하는 한 규율로는 못 막는다 (M2). 따라서 공식 경로 자체를 3초에 수렴시킨다.

3. **오염 유입점의 기술적 봉쇄 (Block, Don't Warn)**
   - 원문 수정이 비원천 위치에서 발생하는 순간을 **경고가 아닌 거부**로 차단한다 (권한 강등, CI validator 실패, 단방향 sync).
   - 탐지만 하고 승인받는 경로는 반복 실패한다 — 알림은 무시되고, 급할 때는 우회된다.
   - **변경 (v1→v2) 강화 절 P3-a**: **차단의 시간 창(window)은 0초여야 한다. 편법 행위와 원복 사이 임의 N분 지연이 있는 설계는 '탐지+복구'이며 본 원칙에서 차단으로 분류하지 않는다.** "자동 복구"·"주기적 cron 덮어쓰기"·"N분 뒤 sync"는 모두 탐지형으로 계수한다. Architect가 "10분 후 복귀"를 "차단"으로 명명해도 Critic 평가 시 탐지로 재분류되며, iteration-contract v2 감점 트리거에 해당한다.

4. **파생물 재생성 원칙 (Derivatives Are Disposable)**
   - `.pb`, Lokalise 내 한국어, dictionary 캐시는 **언제든 원천에서 재생성 가능한 파생물**로만 존재한다. 손으로 고친 파생물은 다음 빌드에서 자동 덮어쓰여야 정상.
   - rev 누적 / 바이너리 잔여물 / unpublished 잔류는 "파생물을 원천처럼 다뤄서" 발생 — 재생성 가능성을 매 빌드 검증.

5. **외부 법인 경계 분리 (Boundary by Default)**
   - VNG 등 외부 편집 권한자는 **별도 네임스페이스/프로젝트**로 격리되어 메인 원천과 직접 쓰기 충돌이 불가능해야 한다.
   - 04-17 "Indonésia" 사건처럼 메인과 VNG 값이 한 슬롯에서 섞이는 구조 자체를 제거.

6. **점진 전환, 계약 기반 이행 (Contract-Driven Migration)**
   - 고정 제약(Lokalise 구독, VNG 권한, 번역팀 UI, Excel 친숙도)을 **동시에 뒤집지 않는다**. 원칙 1~5를 각 단계별 계약으로 쪼개 Phase 0~5에 배치.
   - 한 번에 바꾸면 4~6주 내 불가능하고, 실패 시 롤백 비용이 구독 해지급으로 커진다.

- **Step 4**: 원칙 1~6을 만족하는 후보안 **5개 이상** 도출 (서로 다른 축: 도구 교체 / 권한 강등 / 단방향 동기화 / 네임스페이스 분리 / 하이브리드 등).
- **Step 5**: 각 후보안에 대해 **편법 경로 재현 시나리오**와 **차단 여부**를 5건 사례(02-05 / 02-20 / 03-07 / 03-11 / 04-17)별로 명시. 편법 경로 매트릭스는 iteration-contract v2에 따라 **8종 × 5+후보**로 확장.
- **Step 6**: 6축 평가 매트릭스 — (고정 제약 준수도 / 5건 차단률 / 개발 공수 4~6주 적합도 / 번역팀 UI 학습비용 / VNG 충격도 / 롤백 용이성). 각 축 최대 10점 기준.
- **Step 7**: 추천안 1~2개 + Phase 0~5 전환 경로 (각 Phase의 엔트리·이그짓 크라이테리아 포함). Step 7 본문 작업 목록과 시뮬레이션 표의 **일관성 검증 필수** (iteration-contract v2 DoD 1번 항목).

### 건드리지 않을 영역 (Planner 권한)

- **메타-원인 재정의 금지** — M1/M2는 v1 확정, v2에서 각주만 추가. Architect가 증상 단위로 재분해 불가.
- **해결 원칙 재작성 금지** — 원칙 1~6은 v1 확정. P3에 0초 절이 추가되었고 그대로 구현 대상이다. Architect는 원칙을 **구현**할 뿐 원칙을 바꾸지 않음.
- **고정 제약 완화 금지** — Lokalise 구독 해지, VNG 권한 회수, Excel→Google Sheet 일괄 전환 등은 후보안의 **전제**로 쓸 수 없음 (Phase 5 이후 선택지로만).
- **특정 도구 편애 금지** — `google_sheet_workflow.md`는 후보 중 하나. "Google Sheet가 답"으로 전제하고 나머지 4개를 들러리로 만드는 후보 설계 금지.
- **규율 의존 금지** — "교육·주의·혼내기·합의·공지·주기적 리뷰" 같은 단어가 차단 메커니즘으로 등장하면 critic 감점.
- **차단 시간 창 정의 금지 (v1→v2 신규)** — 원칙 P3의 "0초 절"을 Architect가 "10분 창을 차단으로 재분류"하는 식으로 완화할 수 없음. 원복 지연이 1분 이상이면 Critic이 자동으로 탐지로 재분류한다.
- **M2 해석 범위 축소 금지 (v1→v2 신규)** — "수동 단계 0 = M2 만족"으로 해석 축소 불가. 공식 경로 종단 시간이 3초 수렴이 아니면 M2 미충족으로 계수.
