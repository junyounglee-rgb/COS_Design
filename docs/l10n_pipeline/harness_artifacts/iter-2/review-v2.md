## 7. 남은 약점 (v2에서도 해결 안 된 것) — v3 숙제

### 7.1 M2 "3초 수렴"의 부분 달성

- **현상**: L451 Slack 커맨드 "30초 SLA" + L461 "종단 시간 < 3초"는 Slack 슬래시 **응답 시간**. 자동 PR → CI 머지까지는 30초. 플레이어에게 배포 반영은 CI/CD 파이프라인 시간(분 단위) 추가.
- **재현 시나리오**: 지희가 Slack `/ko-typo <str_id> <수정후>` 입력 → 3초 내 Slack 응답 → 30초 후 PR 머지 → 다음 배포(분~시간) 후 플레이어 반영. "공식 경로 종단 3초"(spec L55~56 원문)는 Slack 응답에 국한.
- **v3 숙제**: spec v2 [M2-a]의 "종단 시간"이 **플레이어 반영까지**인지 **소스 변경 머지까지**인지 Planner가 명확화해야 함. 현재는 양쪽 해석이 가능.

### 7.2 Phase 0 리드타임 비현실성

- **현상**: L395 "Phase 0 Week 0, 3~5일"에 VNG 권한 회수 + Slack 창구 개설 + 7일치 측정 + cos-vng 생성 킥오프 + 담당자 지정 5종 동시 수행. VNG 협의는 법인간 조율로 3~5일로는 짧음.
- **재현 시나리오**: VNG가 과도기 Slack 창구 동의를 4주 지연 → Phase 0 Exit 지연 → Phase 1~4 전체 일정 밀림 → 4~6주 고정 제약 위반.
- **v3 숙제**: Phase -1(사전 VNG 조율) 추가 또는 Phase 0 기간을 1~2주로 현실화.

### 7.3 Slack 대리 입력 창구의 지속성

- **현상**: L386 "주 10~15건 예상"을 기획팀 담당자 1명에게 Phase 0~3 기간(4주 이상) 지속. 주말·야간·휴가 대응 미정의.
- **재현 시나리오**: 담당자 휴가 시 대체자 미지정 → VNG 요청 적체 → VNG가 개인 API 토큰 발급 시도(F2 차단되지만 운영 압박 증가) 또는 기존 경로 우회 시도.
- **v3 숙제**: Slack 창구를 "기획팀 당번제(2인)" 또는 "자동 폼 + SLA 24시간"으로 구조화.
