## 완료의 정의 (Definition of Done)

- [ ] **후보안 5개 이상** (서로 다른 축에서 출발)
  - 권장 축 예시: (A) 도구 교체형 / (B) 권한·경로 강등형 / (C) 단방향 동기화형 / (D) 네임스페이스 분리형 / (E) 하이브리드 점진형
  - 동일 축 변형(예: Google Sheet A / Google Sheet B)은 1개로 카운트
- [ ] **각 후보에 편법 경로 + 차단 여부 명시**
  - 편법 경로 목록(최소 5종): ① Lokalise 웹 한국어 직수정 / ② 대표키 병합 옵션 실수 / ③ `.pb` 직접 편집 커밋 / ④ git 커밋 누락 상태로 로컬 datasheet 실행 / ⑤ VNG 편집값이 메인에 역류
  - 각 후보 × 각 편법 = "차단됨 / 탐지됨 / 통과됨" 3단계로 명시
- [ ] **평가 매트릭스 6축 점수** (각 0~5점, 총점 30점 만점)
  - (1) 고정 제약 준수도 — Lokalise 구독/VNG 권한/Protobuf/Excel 친숙도/번역팀 UI/개발 4~6주 모두 만족하는가
  - (2) 5건 사례 차단률 — 02-05, 02-20, 03-07, 03-11, 04-17 재현 시 몇 건이 구조적으로 차단되는가
  - (3) 개발 공수 적합도 — 1~2명 × 4~6주 안에 완성 가능한가
  - (4) 번역팀 UI 학습비용 — 번역팀이 Lokalise 외 신규 UI 학습 필요한가
  - (5) VNG 충격도 — VNG 법인 현업 작업 흐름 중단 리스크
  - (6) 롤백 용이성 — 실패 시 원복 가능 여부 및 소요
- [ ] **추천안 1~2개** (단일 정답 강요 금지)
  - 추천 이유는 "편법 차단률 + 고정 제약 준수"의 파레토 프론티어 관점에서 설명
  - 2개를 낼 경우 선택 기준(예: 공격적 전환 vs 보수적 전환)을 명시
- [ ] **전환 경로 Phase 0~5**
  - 각 Phase마다 **Entry Criteria** (시작 조건) + **Exit Criteria** (완료 조건) + **롤백 포인트** 명시
  - Phase 0 = 사전 준비 (관찰·측정), Phase 5 = 고정 제약 완화 선택지 (예: Lokalise 구독 종료 재검토)
  - 각 Phase 예상 기간을 4~6주 총합 안에 맞춰 배치

## 통과 기준

### 필수 (Architect 제출 → critic 평가 전 자가 점검)

- [ ] 후보안 5개 이상 제출
- [ ] 각 후보 × 5종 편법 경로 매트릭스 완성 (5 × 5 = 25셀)
- [ ] 6축 평가 매트릭스 점수표 완성
- [ ] 추천안 1~2개 선정 근거 서술
- [ ] Phase 0~5 각각의 Entry/Exit Criteria + 롤백 포인트 명시

### 합격 (critic 평가)

- **critic 총점 >= 24/30점** (6축 × 5점 기준, 80%)
- **5건 사례 재현 시뮬레이션 전부 "차단됨"** — 추천안이 02-05 / 02-20 / 03-07 / 03-11 / 04-17 모두를 구조적으로 차단 (탐지만으로는 불가)
- **메타-원인 커버리지** — 추천안이 M1(단일 쓰기 원천)과 M2(편법 비용 역전)를 **둘 다** 만족 (하나만 만족하면 불합격)
- **고정 제약 6종 준수** — Lokalise 구독 / VNG 권한 / Protobuf / Excel 친숙도 / 번역팀 UI / 개발 4~6주 **전부** 충족 (1개라도 위반 시 불합격)

### 감점 트리거

- "교육", "주의", "혼내기", "합의", "공지", "주기적 리뷰" 등 **규율 의존 단어**가 차단 메커니즘으로 등장 → -3점/건
- "Google Sheet가 유일한 답" 식 전제 → -5점
- Phase 0~5 중 롤백 포인트 누락 Phase → -2점/건
- 5건 사례 중 1건이라도 "탐지됨"으로 분류되면 합격 불가 (0점 처리)
