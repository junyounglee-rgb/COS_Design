## 주요 발견 사항

### @st.cache_data와 언더스코어 파라미터

- Streamlit `@st.cache_data`는 `_`로 시작하는 파라미터를 캐시 키에서 제외
- `_local_mtime` 파라미터 → 파일 변경 시 캐시 무효화 안 됨
- 수정: `local_mtime`으로 이름 변경 (언더스코어 제거)

### Windows bat 파일 인코딩

- Python Write 도구는 UTF-8로 파일 저장
- Windows `cmd.exe`는 bat 파일을 ANSI(CP949)로 파싱 → 한국어 포함 시 파싱 실패
- `chcp 65001`은 출력 코드페이지만 변경, bat 파일 파싱에는 영향 없음
- 수정: bat 파일 내 한국어 전부 영어로 교체

### install.bat .venv 미생성 문제

- PowerShell의 `cmd /c` 실행 시 bat 파일 인코딩 에러로 venv 생성 단계 실패
- 실제 사용 시 직접 더블클릭으로 실행 필요
- bat 영어화 후 정상 동작 확인

### Excel 헤더 행 위치 (게임 데이터 특수성)

- 대부분의 게임 데이터 시트는 row 1-2가 주석·경로 메타, **row 3에 실제 컬럼 헤더** (`$filter`, `^key`, `category`, `#설명` 등)
- 기존 diff 로직은 row 1을 헤더로 가정 → 비교 결과에서 컬럼명이 `col1`, `col2`처럼 일반 이름으로 표시
- 시트명에 `#`이 있는 시트는 주석용 시트 패턴으로 row 1이 헤더
- `_detect_header_row()` 도입으로 시트별 헤더 행 자동 감지, `_align_rows()`는 `r > header_row` 조건으로 메타 행 전부 제외

### flat dict 행 스캔 병목 (compare_xlsx_side_by_side)

- `_read_sheet_data()` 반환값은 `dict[(row, col), str]` 형태의 flat dict
- `compare_xlsx_side_by_side()`와 `_align_rows()`가 특정 행만 뽑기 위해 `{c: v for (r, c), v in data.items() if r == target}` 패턴을 반복 사용 → 행 N개, 셀 M개일 때 O(N×M)
- `dialog_groups.xlsx` (3시트, 10만 셀) self-vs-self 비교 시 23초 소요
- 시트 루프 진입 시 `_index_by_row()` 로 1회 재인덱싱 (`{r: {c: v}}`) → 이후 행 조회 O(1)
- 최적화 후 1.1초 (22× 개선), 실제 커밋 diff 정확성 동일 확인

### diff_core.py ↔ cherry_pick.py 코드 중복

- 초기 분리 시 `cherry_pick.py`에 diff 함수 7개가 그대로 남음 (중복 ≈ 350줄)
- 헤더 감지 수정 시 양쪽 모두 손대야 했던 것이 문제 재인식의 계기
- `cherry_pick.py`가 `sys.path.insert`로 `excel_diff/` 경로 추가 후 `from diff_core import ...` 방식 채택
- 공유 심볼: `run_git_binary`, `_read_sheet_data`, `_detect_header_row`, `_get_headers`, `_align_rows`, `_build_comparison_html`, `compare_xlsx_side_by_side`
- cherry_pick 버전이 보유하던 `syncTableSize()` JS 개선은 `diff_core`로 역전파

## 환경 이슈 및 해결

| 이슈 | 원인 | 해결 |
|------|------|------|
| cmd 창 뜨자마자 꺼짐 | bat 파일 UTF-8 한국어 → `cmd.exe` 파싱 실패 | `run.bat`/`install.bat` 영어 전용으로 재작성 |
| `_local_mtime` 캐시 무효화 안됨 | Streamlit `_` 접두사 파라미터 캐시 키 제외 규칙 | 파라미터명 `local_mtime`으로 변경 |
| `install.bat` `.venv` 미생성 | PowerShell 통한 bat 실행 시 인코딩 에러 | 영어화 후 해결, 직접 더블클릭 권장 |
| git log 한국어 깨짐 | PowerShell 콘솔 CP949 인코딩 | Streamlit 브라우저에서는 정상 (UTF-8 렌더링) |
| diff 표에서 컬럼명이 `col1`·`col2`처럼 표시 | row 1을 헤더로 가정, 실제 헤더는 row 3 | `_detect_header_row()` 도입 — 시트명 `#` 포함→1, 그 외→3 |
| 동일한 diff 로직이 두 파일에 중복 | 초기 분리 후 공유 구조 미정립 | `cherry_pick.py`가 `diff_core.py`를 import하도록 전환 (-328줄) |
| 스크롤 중 좌우 테이블 높이 불일치 | `syncHeights()`만으로는 `minHeight`·`minWidth` 동기화 부족 | `syncTableSize()` 함수 추가 |
| 원격 브랜치 목록이 빈 리스트로 반환되면 원인 불명 | stderr 수집 없이 silent fail | `get_remote_branches()` 반환 타입을 `(list, err)` 튜플로 변경, `st.warning` 표시 |
| 커밋 비교가 20초 이상 걸림 | flat dict `data.items()`를 행마다 전체 순회 → O(N×M) | `_index_by_row()` 로 `{r:{c:v}}` 1회 재인덱싱, 이후 O(1) 조회 (23,166 → 1,077 ms) |
